# Security Policy

## Reporting A Vulnerability

Please report suspected vulnerabilities privately by emailing
`security@dauntlessagentic.com`. If that address is not yet active, use
`craig@dauntlessagentic.com` and include `Security disclosure` in the subject.

Include enough detail to reproduce the issue:

- the affected URL, route, or package;
- steps to reproduce;
- observed impact;
- any proof-of-concept code or screenshots that are safe to share.

Do not open public GitHub issues for security reports.

## Scope

In scope:

- `dauntlessagentic.com` and its public website routes;
- `/portal` demo or client portal routes;
- `/api/portal/v1/*` routes;
- repository code and configuration that could affect production deployment.

Out of scope:

- denial-of-service testing;
- social engineering;
- physical attacks;
- attacks against third-party services not controlled by Dauntless Agentic.

## Response Targets

We aim to acknowledge reports within 3 business days, confirm scope and severity
within 10 business days, and provide remediation status as fixes ship.

## Safe Harbor

Good-faith research that stays within this policy, avoids data destruction, and
does not access or disclose third-party data will not be pursued by Dauntless
Agentic. Stop testing and contact us immediately if you encounter real client
data, credentials, or non-public personal information.
